# CLAUDE.md

- Shelly `shelly-diy` OAuth returns the JWT access token directly as the `code` param — no token exchange needed.
- On-device mJS script limits (measured): per-call `Script.PutCode` POST body ≤ 8192 B (HTTP 413 above); per-script size ≤ 65535 B (`code -103 "Script length exceeded 65535 bytes limit!"`); max 10 script slots total (`code -108`). Per-script cap is per-slot, not aggregate. `shellyctl script upload` chunks PutCode automatically — raw sources under 64 KiB work without `--minify`. Runtime: mJS heap ≈ 2 KB per eval frame, concurrent `Shelly.call()` cap ≈ 5 (fans out `"Too many calls in progress"` above); serialize RPC chains + split large payloads into small batches. `Array.prototype.concat` isn't implemented — write a manual helper. **KVS value size ≤ 254 bytes** (Gen4 PowerStrip: `code -103 "length should be less than 255!"`). Use `KVS.Set`/`KVS.Get` via `Shelly.call()` (not as globals); `Shelly.emitEvent()` + `Shelly.addEventHandler()` provide cross-script IPC on the same device (works on Gen3/Gen4; doesn't trigger phone push notifications — use `HTTP.GET` to a Cloud Scene for that).
- The PowerStrip has 4 switches; cloud server is `shelly-96-eu.shelly.cloud`.
- Cloud `scene/add` API is undocumented. Mandatory fields in `scene_script`: `_enabled: true`, `_run_on_ingest: true`, `if: {"or":[{"and":[]}]}` (empty condition), and `do` array with `notify: "push_notification"`. Missing any of these creates ghost scenes that break the app.
